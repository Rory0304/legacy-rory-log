When to use Server and Client Components? 서버 사이드 백엔드 데이터를 패치하는 경우 사이즈가 큰 dependency 를 사용하는 경우 클라이언트 사이드 onClick(), onChange() 와 같은 이벤트 리스너를 사용하는 경우 state 관리를 하거나 react life cycle 을 사용하는 경우 여기서 중요한 점은 Client Component 에서 서버에서 동작하는 로직이 포함된 Server Component 를 임포트할 수는 없다. 서버 컴포넌트를 브라우저 단에서 실행할 수는 없기 때문이다. 'use client'; // This pattern will **not** work! // You cannot import a Server Component into a Client Component. import ExampleServerComponent from './example-server-component'; export default function ExampleClientComponent({ children, }: { children: React.ReactNode; }) { const [count, setCount] = useState(0); return ( &lt;&gt; &lt;button onClick={() =&gt; setCount(count + 1)}&gt;{count}&lt;/button&gt; &lt;ExampleServerComponent /&gt; &lt;/&gt; ); } 다만, 클라이언트 컴포넌트에서 서버 컴포넌트를 children props 으로 받을 수는 있다. // This pattern works: // You can pass a Server Component as a child or prop of a // Client Component. import ExampleClientComponent from './example-client-component'; import ExampleServerComponent from './example-server-component'; // Pages in Next.js are Server Components by default export default function Page() { return ( &lt;ExampleClientComponent&gt; &lt;ExampleServerComponent /&gt; &lt;/ExampleClientComponent&gt; ); } 이는 컴포넌트 합성과 관련이 있는데, 위 코드를 참고하여 이해해보자면, 서버 사이드로 동작하는 Page.tsx 에서 클라이언트 사이드의 컴포넌트 ExampleClientComponent를 임포트해오고 ExampleClientComponent는 서버 사이드의 컴포넌트인 ExampleServerComponent를 children 으로 받는다. React will know it needs to render on the server before sending the result (which doesn't contain any server-only code) to the client. From the Client Component's perspective, its child will be already rendered. 즉, 클라이언트 컴포넌트는 ReactNode 를 prop 으로 받을 수 있고, ReactNode 는 서버 컴포넌트에 의해 렌더링할 수 있기 때문에 서버 컴포넌트를 children prop 으로 전달하는 것은 가능하다. 클라이언트 컴포넌트의 관점에서는 child 인 서버 컴포넌트가 이미 렌더링되었기 때문이다. 3) Data Fetching 13 버전에서는 getStaticProps / getInitialProps / getServersideProps 의 방법은 지원되지 않고 fetch option 으로 대체되었다. // This request should be cached until manually invalidated. // Similar to `getStaticProps`. // `force-cache` is the default and can be omitted. fetch(URL, { cache: 'force-cache' }); // This request should be refetched on every request. // Similar to `getServerSideProps`. fetch(URL, { cache: 'no-store' }); // This request should be cached with a lifetime of 10 seconds. // Similar to `getStaticProps` with the `revalidate` option. fetch(URL, { next: { revalidate: 10 } }); { cache: 'force-cache' }&nbsp;- getStaticProps 처럼 동작하며, 기본값이다.     ES6 import / export 기본 사용 방법

    시작에 앞서, React 에서 모듈간 코드 공유와 재사용을 위해 자주 사용되는 import / export ES6 구문에 대해 알아보고자 한다. 쉽고 간단할 수록 , ‘하던 대로’ 코드를 작성하게 된다. 이번 포스팅을 통해, 조금 더 의식적으로 코드를 작성해보자. 모듈이란? 코드의 재사용성을 위해 사용되는 독립적인 코드의 단위이다. function / component 등을 담을 수 있고, 다른 파일에서 임포트해서 사용할 수도 있다. 여기서 “모듈화”를 한다고 하면, 코드를 독립적인 단위로 분리해서 구성하는 것을 의미한다. export와 import 사용 방법 Default Export / import 단일 모듈을 내보낼 때 사용된다. 이때, 한 파일 내에서 export default 키워드는 오직 하나만 있어야 한다. const myFunction = () =&gt; { /* 함수 내용 */ }; export default myFunction; default export 를 사용한 모듈을 import 해 올 때는 다른 이름으로 정의하여 사용할 수 있다. import myFunction from './myFunction'; // 다른 이름으로 정의하여 import 해올 수 있다. import myFunc from './myFunction'; 단, let / const 를 바로 내보낼 수는 없다. function / class 는 가능하다. export default let myValue1 = 1; // 불가 export default const myValue2 = 2 // 불가 export default function myFunction () { ... } // 가능 export default class myClass extends React.Component {} // 가능 Named Export / import 한 파일에서 여러 모듈을 내보낼 때 사용한다. 내보내기할 대상 앞에 export 키워드를 사용한다. 하나씩 내보내기 export const myFunction = () =&gt; { /* 함수 내용 */ }; export const myVariable = 1 export class MyClass { /* 클래스 내용 */ } 목록으로 내보내기 export { myFunction, myVariable, MyClass }; 내보내면서 이름을 바꾸기 export { myFunction as myFunc, myVariable as myVar}; 비구조화로 내보내기 export const { name1, name2: bar } = o; Default + Named export import 두 방식을 섞어서 사용할 수도 있다. (이 방식은 아직까지 사용해본 적은 없다) function example1() { console.log("example1"); } function example2() { console.log("example3"); } function example3() { console.log("example3"); } export { example1 as default, example2, example3 }; 파일 구성과 모듈 조합 현업에서는 하나의 목적을 가진 무수한 모듈들을 임포트하여 사용해야 하는 상황이 발생하는데, 이때 적절히 파일 구성을 해주어야, 코드 재사용성을 높일 수 있다. 예를 들어, 아래 사진처럼 ‘components’ 폴더 안에 ‘blocks’ 의 목적을 가진 여러 모듈 (Box, Hspace, Vspace)이 위치한다고 가정하고 page 컴포넌트에서 blocks 들을 가져와 사용하는 상황이라고 해보자. 첫번째로 default export / import 를 사용하여 구성하는 방법이다. // Box.tsx const Box: React.FC = () =&gt; &lt;div&gt;...&lt;/div&gt; export default Box; // page.tsx import Box from 'src/components/blocks/Box'; import Vsapce from 'src/components/blocks/Vsapce'; import Hspace from 'src/components/blocks/Hspace'; 이 방법의 예상되는 단점은 다음과 같다. 모듈 사용부에서 중복 경로를 매번 작성해주어야 함 import 시 변수명을 자유롭게 설정할 수 있기 때문에 개발자에 따라 명명 방법이 달라질 수 있음 ⇒ 리팩토링 비용 증가로 이어짐 이를 보완해주려면, 모듈을 조합하는 방식을 사용하면 된다. 모듈 조합은 export 된 모듈들을 모아서 한 번에 re-export 하는 방식이다. 즉, 아래처럼 Blocks 경로에 있는 Box, Vspace, Hspace 컴포넌트를 한 번에 불러올 수 있다. import { Box, Vspace, Hspace } from 'src/compoentns/blocks' 모듈 조합을 적절히 사용한다면, 위 코드 처럼 중복 코드를 제거하여 필요한 모듈만을 불러올 수 있을 것이고 미리 정의된 이름을 사용하여 리팩토링 비용을 낮출 수 있을 것이다. 모듈 조합을 위해서는 index 파일을 사용한다. ES6에서 index 파일의 역할은 이 링크에서 자세히 알아볼 수 있다. In&nbsp;ES6, having an&nbsp;index.js&nbsp;file in a folder lets you perform an&nbsp;import&nbsp;from the folder implicitly without specifying the&nbsp;index.js&nbsp;in the import statement – just like how web servers will serve up the&nbsp;index.html&nbsp;in a folder without you needing to explicitly put the&nbsp;index.html&nbsp;in the URL. 먼저 blocks 경로 하위에 index.ts 파일을 생성하고 export 된 모듈들을 다시 re-export 해준다. 이때 re-export 해주는 방식은 크게 2가지 이다. defined export 를 사용했을 경우 // Box.tsx const Box: React.FC = () =&gt; &lt;div&gt;...&lt;/div&gt; export default Box; ‘as’ 키워드를 활용하여, 정의된 이름으로 모듈을 import 해올 수 있도록 한다. // components/blocks/index.ts export { default as Box} from "./Box"; export { default as Hspace } from "./Hspace"; export { default as Vspace} from "./Vspace"; named export 를 사용했을 경우 // Box.tsx export const Box: React.FC = () =&gt; &lt;&gt;...&lt;/&gt;; export const Flex: React.FC = () = // Hspace exoort const Hspace : React.FC = () =&gt; &lt;&gt; ...&lt;/&gt; ‘*’ 를 사용하여 named export 를 모두 내보낼 수 있다. // components/blocks/index.ts export * from "./Box"; // Box 와 Flex 컴포넌트가 모두 내보내진다. export * from "./Hspace"; export * from "./Vspace"; 만약, 특정 모듈만을 pick하여 export 를 하고 싶다면 아래와 같이 작성해주면 된다. export { Box } from "./Box"; export * from "./Hspace"; export * from "./Vspace"; 참고 자료 https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Statements/export https://www.digitalocean.com/community/tutorials/react-index-js-public-interfaces https://ko.javascript.info/import-export https://jamesknelson.com/re-exporting-es6-modules/
